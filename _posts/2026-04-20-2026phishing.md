# Phishing in 2026: What Changed?<br>
 
ClickFix, OAuth & AitM, BitB, PhaaS & AI for sure.<br>

Phishing is no longer just about fake emails and login pages.<br>

In 2025–2026, it has evolved into a highly industrialized, AI-powered ecosystem capable of bypassing traditional defenses, MFA, and even user awareness.<br>

As a SOC Analyst, if you’re still thinking in terms of “suspicious links and attachments,” you’re already behind.<br>

## From Deception to Identity Takeover<br>

-   Phishing evolved from **email scams → identity takeover operations**<br>
-   Attackers now target **sessions, tokens, and user behavior**<br>
-   **AI + PhaaS** made attacks scalable and accessible<br>
-   **MFA is no longer enough** due to session hijacking<br>
-   New techniques:<br>
    -   ClickFix (user executes attack)<br>
    -   OAuth abuse (no password needed)<br>
    -   BitB (fake login windows)<br>
    -   SEO phishing (no email involved)<br>

# Phishing Is Bigger Than Ever<br>
Phishing is no longer a tactic, it’s an **industry**:<br>
-   Over 3.8 million phishing attacks recorded in 2025<br>
-   Around 3.4 billion phishing emails sent daily<br>
-   AI-driven phishing increased success rates by up to 400%<br>
-   Phishing kits expected to drive 90%+ of credential theft by 2026<br>


# 1. AI-Generated Phishing<br>

Attackers now use AI as a default tool.<br>

### What changed:<br>

-   No more broken English<br>
-   Context-aware, personalized messages<br>
-   Campaigns generated in minutes<br>

### Impact:<br>

-   ~60% higher click rates<br>
-   82%+ of phishing emails show AI involvement<br>

Grammar-based detection is obsolete.<br>

# 2. Phishing-as-a-Service (PhaaS)<br>

Phishing is now plug-and-play.<br>

### Capabilities:<br>

-   MFA bypass<br>
-   CAPTCHA evasion<br>
-   Traffic filtering<br>
-   Anti-analysis logic<br>

### PhaaS Examples:<br>

W3LL, Tycoon 2FA, EvilProxy<br>

That said, Low-skill attackers can run advanced campaigns.<br>

# 3. MFA Bypass & Adversary-in-the-Middle (AitM)<br>

### How it works:<br>

-   Victim logs into a proxy phishing page<br>
-   Credentials forwarded to real service<br>
-   Session cookies stolen in real time<br>

### Result:<br>

-   MFA becomes ineffective<br>
-   Session hijacking replaces password theft<br>

Identity sessions are the real target.<br>

# 4. Multi-Channel Phishing<br>

Phishing is no longer email-centric.<br>

### Channels:<br>

-	Callback phishing<br>
-	Calendar invites<br>
-	Smishing (SMS)<br>
-	Vishing (voice, +449% growth)<br>

Attackers go where visibility is weakest.<br>

# 5. Payload Evasion Techniques<br>

### Modern tricks:<br>

-   QR phishing (Quishing)<br>
-   Password-protected files<br>
-   SVG payloads<br>
-   Image-based phishing<br>
-   Trusted service link masking<br>

Static detection is failing.<br>

# 6. Polymorphic & Dynamic Phishing<br>

### Characteristics:<br>

-   Unique URLs per victim<br>
-   Dynamic email content<br>
-   Infrastructure rotates every ~12 hours<br>

Signature-based detection is collapsing.<br>

# 7. Deepfake & Identity Attacks<br>

### Techniques:<br>

-   Deepfake voice (CEO fraud)<br>
-   AI video impersonation<br>
-   Real-time identity spoofing<br>

Trust itself is now the attack surface.<br>

# 8. Abuse of Legitimate Platforms<br>

70% increase in attacks from trusted platforms<br>

### Common platforms:<br>
-   Microsoft 365<br>
-   Google Docs<br>
-   SharePoint<br>
-   Dropbox<br>

### Impact:<br>
Bypasses security and user suspicion<br>

# 9. ClickFix Attacks (User-Assisted Execution)<br>

One of the most dangerous recent evolutions. Attackers combines social engineering + Living-off-the-Land Binaries. **They are forcing Users to execute the attack themselves**.<br>

### How it works:<br>

Users are tricked into:<br>

-   Opening Run (Win + R)<br>
-   Pasting malicious commands<br>
-   Executing PowerShell or scripts<br>

### Result:<br>

-   No exploit needed<br>
-   No malicious attachment<br>
-   User executes the attack<br>

# 10. Browser-in-the-Browser (BitB)<br>

### What happens:<br>

-   Fake SSO popups inside the browser<br>
-   Pixel-perfect login windows<br>
-   Fully interactive phishing UI<br>

### Impact:<br>
- Users cannot distinguish real vs fake<br>
- “Check the URL” no longer works<br>

# 11. OAuth Consent Phishing<br>

### Key idea:<br>

No credentials needed because this is **token abuse**, not credential theft.<br>

### Flow:<br>

-   User grants permissions to malicious app<br>
-   Attacker gains API access<br>

### Impact:<br>

-   No MFA triggered<br>
-   Persistent access even after password reset<br>

# 12. SEO & Ad-Based Phishing<br>

In this technique, users find phishing pages themselves. No email involved.<br>

### Tactics:<br>

-   Malicious ads<br>
-   Fake download portals<br>
-   SEO poisoning<br>

# 13. Session Replay & Token Abuse<br>

Attackers own the session, not the password.<br>

### New behaviors:<br>

-   Session reuse across locations<br>
-   Refresh token theft<br>
-   Long-lived access<br>

# 14. Phishing + Malware Convergence<br>

Phishing is now a full attack chain.<br>

### Modern campaigns:<br>

-   Deliver infostealers<br>
-   Drop loaders<br>
-   Combine identity + endpoint compromise<br>

# 15. HTML Smuggling<br>

Bypassing network and email inspection.<br>

### Technique:<br>

-   Payload built locally in browser<br>
-	No remote/external payloads<br>
-   Encrypted data reconstructed via JavaScript<br>

# Key Takeaways for SOC Teams<br>

-   Phishing = Identity attack, not email threat<br>
-   MFA is no longer sufficient<br>
-   AI is scaling attacks dramatically<br>
-   Users are now part of the attack chain<br>
-   Session hijacking > credential theft<br>

# Detection Strategy (Modern SOC)<br>

Defenders should correlate events across **Identity**, **Endpoint** & **Network** layers.<br>

### Identity Layer:<br>

-   Token usage & reuse<br>
-   OAuth activity<br>
-   Session anomalies<br>

### Endpoint Layer:<br>

-   Command-line execution<br>
-   LOLBins (PowerShell, mshta)<br>
-   Browser spawning processes<br>

### Network Layer:<br>

-   Short-lived domains<br>
-   Suspicious outbound auth flows<br>

### User Behavior:<br>

-   Manual command execution<br>
-   Abnormal interaction patterns<br>
